# 반 명단 기반 문서 자동화 웹앱 설계

## 배경

기존에는 `first toeic example.xlsx` / `test.xlsx` 형태의 파일 하나에 전체명단·인쇄용 출석부·온라인 출석부·관리일지·교재배부명단 탭이 모두 들어있었고, `AI_WORKFLOW_PROMPT.md`에 정리된 규칙대로 전체명단 탭을 기준으로 나머지 탭을 채웠다. 이 워크플로우를 여러 반 선생님이 쓸 수 있는 웹 도구로 옮긴다. 단어 시험 채점 기능은 별도 설계 문서([2026-07-03-vocab-grading-web-design.md](2026-07-03-vocab-grading-web-design.md))에서 다룬다.

## 핵심 제약

1. 개인정보(학번, 학과, 연락처 등)는 서버 어디에도 저장하지 않는다. 요청 처리 중 메모리에만 존재하고 응답 후 폐기한다.
2. 업로드 양식(전체명단 탭의 컬럼 구조)은 지금 쓰는 양식과 동일하다고 가정한다.
3. 생성되는 문서(출석부/교재배부명단/관리일지)의 컬럼 구성은 기존 시트 탭과 완전히 같아야 한다.
4. 없는 정보나 사실을 추가하지 않는다 — 전체명단에 없는 학생을 임의로 만들어 넣지 않는다.
5. 결과는 정확해야 하고, 검토 가능해야 한다.
6. 결과물은 복사/붙여넣기로 쓸 것이므로 붙여넣기 편한 형태로 제공한다.

## 아키텍처

- 단어 시험 채점 기능과 같은 Next.js/Vercel 프로젝트 안에 별도 라우트로 둔다.
- AI 호출이 필요 없다 (전부 규칙 기반 데이터 가공). OpenAI 비용이 들지 않는다.
- 업로드된 전체명단은 서버리스 함수가 요청 처리 중에만 메모리에 두고, 응답 후 버린다. 저장하지 않는다.
- 진입 시 공유 비밀번호로 접근을 제한한다 (기능1과 동일 정책).

## 입력

- 전체명단 탭만 담긴 엑셀/CSV 1개 업로드.
- 구조: 5행 헤더, 6행부터 학생 데이터. 주요 컬럼 — 학번, 이름, 학과, 학년, 전화번호, `1기 신청`/`2기신청`.
- 화면에서 기수(1기/2기)를 선택한다. 실제 헤더가 다를 수 있으므로, 헤더 행에서 "1기"/"2기"가 포함된 신청 컬럼을 찾아 매칭한다.

## 반 분류 규칙 (기존 `AI_WORKFLOW_PROMPT.md` 그대로 적용)

- 토익 반: `기초 오전`, `기초 오후`, `중급 오전`, `중급 오후`, `실전 오전`, `실전 오후`
- 오픽 반: `오픽 중급 오전`, `오픽 중급 오후`, `오픽 실전 오전`, `오픽 실전 오후`
- 모든 반은 오전/오후가 항상 함께 존재하며, 문서 생성 시 오전/오후를 분리해서 처리한다.
- 신청 반 컬럼이 비어 있거나 위 반명과 정확히 일치하지 않으면 불참/제외로 처리하고 어떤 문서에도 포함하지 않는다.
- 전체명단 원본 데이터는 (화면에 표시만 하고) 임의로 학생을 삭제하지 않는다 — 웹에서는 애초에 파일을 수정하지 않고 새 표만 생성하므로 원본 파일 훼손 문제 자체가 없다.

## 화면 구성

1. **비밀번호 입력 화면**: 기능1과 공유.
2. **명단 업로드 화면**: 전체명단 파일 업로드, 기수(1기/2기) 선택.
3. **결과 화면**:
   - 상단에 "전체 반 보기" (기본값) / "반 선택" 드롭다운 토글. 반 선택 시 해당 반(오전+오후)만 필터링해서 보여준다.
   - 반(오전/오후 구분) × 문서 종류 조합으로 표를 생성해 펼쳐서 보여준다.
     - 인쇄용 출석부: `NO, 학과, 이름, 연락처`. 반복 인쇄 블록은 재현하지 않고 깨끗한 표 1개만 생성한다 — 여러 인쇄 블록에 붙여넣는 것은 사용자가 직접 반복한다.
     - 온라인 출석부: 오전 `번호,학년,학번,수강반,성명,연락처` / 오후 동일 구성을 나란히 표시한다.
     - 관리일지: `이름, 학과` 목록만 순서대로 생성한다 (11행 블록 서식 자체는 기존 파일에 이미 있으므로, 웹에서는 순서대로 붙여넣을 이름/학과 데이터만 제공한다).
     - 교재배부명단: `연번, 이름, 학과, 학번, 연락처`.
   - 각 표는 드래그 선택 시 탭 구분자로 복사되어 엑셀에 바로 붙여넣기 가능하게 만든다.
   - 반별 검증 요약을 상단에 표시: "전체명단 해당 반 인원 수 = 생성된 각 문서 인원 수"가 일치하는지 자동 확인하고, 불일치 시 어느 학생이 어디서 빠졌는지 이름 기준으로 표시한다.

## 데이터 흐름

1. 사용자가 전체명단 업로드 + 기수 선택.
2. 서버가 파일을 파싱해 반명 컬럼 기준으로 학생을 분류한다 (오전/오후 포함 전체 반 조합).
3. 반별로 인쇄용 출석부/온라인 출석부/관리일지용 이름학과 목록/교재배부명단 데이터를 계산한다.
4. 반별 인원 수를 원본 전체명단과 대조해 불일치를 계산한다.
5. 결과(표 데이터 + 검증 요약)를 브라우저에 반환한다. 서버는 응답 후 업로드 데이터를 메모리에서 버린다.
6. 사용자가 "전체 보기" 또는 특정 반 선택으로 화면 표시 범위를 좁힐 수 있다 (이미 받은 결과 안에서 클라이언트 필터링, 재요청 없음).

## 오류 처리

- 업로드 파일에 헤더 행(5행)이나 필요한 컬럼이 없으면 "양식이 다릅니다" 안내.
- 기수 신청 컬럼을 못 찾으면 헤더 목록을 보여주고 수동 선택하게 한다.
- 반명이 정의된 10개 조합과 하나도 안 겹치면 "인식된 반이 없습니다" 안내.
- 검증 단계에서 불일치가 발견되면 채점 결과를 막지 않고, 경고와 함께 불일치 학생 목록을 같이 보여준다.

## 테스트 방향

- `test.xlsx`에서 이미 검증했던 사례(박신영 — 신청 컬럼이 비어 있어 제외)를 픽스처로 활용해 제외 로직을 검증한다.
- 반명 매칭(토익 vs 오픽, 오전/오후 구분)에 대한 단위 테스트를 작성한다.
- 인원 수 대조(검증 요약) 로직은 일부러 인원을 어긋나게 만든 픽스처로 불일치 탐지가 되는지 확인한다.

## 범위 밖 (이번 스펙에서 다루지 않음)

- 기존 엑셀 파일의 반복 인쇄 블록·서식을 그대로 재현하는 기능 (웹은 깨끗한 표만 제공).
- 관리일지의 11행 블록 서식 자체 생성 (이름/학과 데이터만 제공, 서식은 기존 파일에 이미 있다고 가정).
- 단어 시험 채점 기능 — 별도 설계 문서에서 다룸.
